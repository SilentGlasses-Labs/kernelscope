# Contributing to KernelScope

Thank you for your interest in contributing to KernelScope! Your contributions help improve transparency and usability for OS analytics.

## How to Contribute

1. **Fork the repository** and clone locally.
2. **Create a new branch** for your feature or bug fix:
```
git checkout -b feature/your-feature-name
```
3. **Make your changes** with clear, descriptive commit messages.
4. **Test your changes** to ensure the dashboard works and scripts run correctly.
5. **Submit a pull request** describing your changes and their purpose.

## Guidelines

- Keep code readable and well-documented.
- Follow the Nord theme and dashboard design conventions.
- When adding new datasets, include source, license, and description.
- Make sure any new scripts are compatible with the existing GitHub Actions workflow.
- Avoid breaking existing functionality.

## Reporting Issues

Please open issues for:

- Bugs or errors
- Suggestions for features or improvements
- Data discrepancies

We value clear, reproducible descriptions for faster resolution.

## Code of Conduct

All contributors are expected to follow the [Contributor Covenant](https://www.contributor-covenant.org/) Code of Conduct.
